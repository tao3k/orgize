** Topology Authority

The project-local category catalog lives in [[file:../topology.toml][docs/topology.toml]].
It follows a path-first contract:

- category semantics come from `docs/topology.toml`;
- note coordinates come from the `AC.ID` prefix in the file path;
- durable note files use `AC.ID_semantic_name.org`;
- the leading category prefix in the directory must agree with the file
  coordinate.

** Authoring Rule

New durable documents should first choose a declared category in
`docs/topology.toml`, then select an unused `AC.ID` slot in that category. If no
existing category is right, propose a reviewable manifest change instead of
silently inventing category meaning.

Markdown remains acceptable only as an ecosystem compatibility shell when
external tooling expects it. Governance and architecture records should be Org
documents under this coordinate system.

After writing documentation, run the local docs lint:

#+begin_src sh
direnv exec . cargo run --quiet -- lint docs development.org
#+end_src
