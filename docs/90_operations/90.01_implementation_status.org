*** M24 Official Org Priority Audit

This audit is based on the local official Org checkout under `.data/org-mode`,
especially `doc/org-manual.org` and the front-line modules in `lisp/`.  The
checkout remains research-only and must not enter Cargo, CI, tests, generated
artifacts, or released packages.

Priority is assigned by common Org usage and by how early/prominent the feature
appears in the official manual and runtime modules:

- P0 means it affects everyday agenda, search, planning, lint, or agent use.
- P1 means it is a popular Org workflow and should be modeled as semantic
      metadata/projection before exporters rely on it.
- P2 means it is important for parity, but should stay opt-in or side-table
      based until the core parser/projection surface is stable.

This is the refreshed backlog table for "which major Org-mode capabilities
should orgize align next".  It differs from the M24 audit above: M24 records
the broad official capability map and historical coverage, while M25 is the

- P0: everyday agenda/search/planning behavior that downstream agents and UIs

      evidence for these, but should not perform the action.
